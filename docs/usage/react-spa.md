# Using boundform with React SPA (Vite, CRA)

## The problem

React apps built with Vite or Create React App are **client-side rendered (CSR)**. When boundform fetches the page via HTTP GET, it receives:

```html
<div id="root"></div>
<script src="/bundle.js"></script>
```

The form HTML is generated by JavaScript at runtime — boundform cannot see it.

## Solution: capture rendered HTML with Playwright

Use Playwright to render the page in a headless browser, then save the HTML for boundform to analyze.

### Step 1: Capture HTML

```bash
mkdir -p boundform/rendered

npx playwright evaluate \
  --url http://localhost:5173/register \
  "document.documentElement.outerHTML" > boundform/rendered/register.html
```

Or use the bundled capture script (created by `npx boundform init`):

```bash
BOUNDFORM_CAPTURE_URL="http://localhost:5173/register" \
BOUNDFORM_CAPTURE_OUTPUT="boundform/rendered/register.html" \
node boundform/rendered/capture.js
```

### Step 2: Create YAML config

Point the `url` to the saved file:

```yaml
# boundform/boundform.yml
pages:
  - url: "boundform/rendered/register.html"
    forms:
      - index: 0
        fields:
          email:
            type: email
            required: true
          password:
            type: password
            required: true
            minlength: 8
```

### Step 3: Validate

```bash
npx boundform --config boundform/boundform.yml
```

## Authenticated SPA pages

Pass cookies via environment variable:

```bash
BOUNDFORM_CAPTURE_URL="http://localhost:5173/settings" \
BOUNDFORM_CAPTURE_OUTPUT="boundform/rendered/settings.html" \
BOUNDFORM_CAPTURE_COOKIE="session=abc123" \
node boundform/rendered/capture.js
```

## Multiple pages

Capture each page separately:

```bash
for page in register login settings; do
  BOUNDFORM_CAPTURE_URL="http://localhost:5173/$page" \
  BOUNDFORM_CAPTURE_OUTPUT="boundform/rendered/$page.html" \
  node boundform/rendered/capture.js
done
```

Then reference all files in a single YAML:

```yaml
pages:
  - url: "boundform/rendered/register.html"
    forms:
      - index: 0
        fields:
          email:
            type: email
            required: true

  - url: "boundform/rendered/login.html"
    forms:
      - index: 0
        fields:
          email:
            type: email
            required: true
          password:
            type: password
            required: true
```

## CI Integration

```yaml
# .github/workflows/form-check.yml
steps:
  - uses: actions/checkout@v4

  - name: Install dependencies
    run: npm ci

  - name: Build and start app
    run: |
      npm run build
      npm run preview &

  - name: Wait for app
    run: npx wait-on http://localhost:5173

  - name: Install Playwright
    run: npx playwright install chromium

  - name: Capture SPA pages
    run: |
      for page in register login; do
        BOUNDFORM_CAPTURE_URL="http://localhost:5173/$page" \
        BOUNDFORM_CAPTURE_OUTPUT="boundform/rendered/$page.html" \
        node boundform/rendered/capture.js
      done

  - name: Validate forms
    run: npx boundform --config boundform/boundform.yml
```

## Also applies to

This guide works for any CSR framework:
- **Vue (Vite)** — same Playwright capture workflow
- **Svelte (Vite, without SvelteKit)** — same approach
- **Angular** — same approach
- **Any SPA** where forms are rendered client-side

## Troubleshooting

### "0 forms found after capture"

- The page might need more time to render. The capture script uses `waitUntil: 'networkidle'` but complex apps may need additional wait time.
- Check if the page requires authentication — provide `BOUNDFORM_CAPTURE_COOKIE`.

### "Fields have no name or id"

This is a limitation of the source HTML, not the capture process. The framework or component library needs to add `name` or `id` attributes to form inputs. See [Prerequisites for your HTML](../../README.md#prerequisites-for-your-html).
